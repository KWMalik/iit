Status Codes

Check out RFC 2616 / Section 10

Status codes are extremely important in responses as they indicate to the client how it should behave up receipt of the response. These codes are very meaningful, as you will come to find out when you start working with RESTful services.

1XXs - Informational
I've still not had to use any code in this branch (100 or 101).
2XXs - Successful
The most common are 200 (OK), 201 (Created), and 204 (No Content).
3XXs - Redirection
Essentially, these codes tell the client something about the resource's state. 301 (Moved Permanently), 302 (Found/Moved Temporarily), and 304 (Not Modified) are common.

4XXs - Client Error
Whoops! The client goofed somehow.

404 (Not Found) is used when the resource cannot be found at all.

400 (Bad Request) generally means there was something wrong the request (or invalid data was passed).

5XXs - Server Error
Something really crazy happened on the server.

500 (Internal Server Error) - you better implement some fault tolerance into your app.
